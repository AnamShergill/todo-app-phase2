# Todo App Project Skills

Here are 10 key skills required for the Todo Full-Stack Web Application project:

1. **Spec-Driven Development**

   * Write, maintain, and interpret structured specifications using Spec-Kit Plus.
   * Ensure all features, API endpoints, and database schemas are accurately described before implementation.

2. **REST API Design & Implementation**

   * Design and implement RESTful APIs with CRUD operations, route validation, error handling, and proper HTTP methods.

3. **User Authentication & JWT Security**

   * Implement authentication flows using Better Auth.
   * Manage JWT issuance, verification, user isolation, and token expiry for secure API access.

4. **Frontend Development with Next.js**

   * Build responsive, interactive, and accessible user interfaces using Next.js (App Router), TypeScript, and Tailwind CSS.
   * Integrate frontend with backend API endpoints using a centralized API client.

5. **Backend Development with FastAPI & SQLModel**

   * Build scalable backend services using Python, FastAPI, and SQLModel ORM.
   * Ensure secure user-specific data operations and database integration.

6. **Database Design & Management (PostgreSQL / Neon)**

   * Design relational schemas, indexes, and constraints in Neon Serverless PostgreSQL.
   * Implement persistent storage for multi-user task data.

7. **UI/UX Design & Theme Development**

   * Create visually appealing, user-friendly, and consistent interfaces.
   * Define color schemes, typography, spacing, responsive layouts, and component states.

8. **Full-Stack Integration**

   * Connect frontend and backend seamlessly, ensuring API responses, authentication, and user interactions work end-to-end.
   * Debug integration issues and optimize data flow.

9. **Testing, QA & Bug Reporting**

   * Conduct thorough testing of frontend, backend, and API endpoints.
   * Ensure security, usability, responsiveness, and spec compliance.

10. **Project & Agent Coordination**

    * Coordinate tasks across multiple agents (spec architect, frontend, backend, QA).
    * Ensure features are implemented according to project phases and specifications, and communicate changes effectively.
